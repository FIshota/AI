# Security Policy

## サポート対象バージョン

ai-chan は現時点で **Phase 2 (PoC 統合段階)** にあり、正式リリース版は存在しません。
`main` ブランチと直近の `phase*/` 系ブランチのみを対象とします。

| ブランチ | セキュリティ修正対象 |
|---|---|
| `main` | ✅ |
| `phase2/*` | ✅ (現行) |
| `phase1/*` | ⚠ 重大脆弱性のみ |
| それ以前 | ❌ |

## 脆弱性の報告

**公開 Issue には絶対に書かないでください。**

### 連絡手段 (優先順)

1. メール: `honnsipittu@gmail.com` (件名に `[SECURITY]` を含める)
2. GitHub Security Advisories (リポジトリが公開された後で有効化)

### 報告に含めてほしい情報

- 影響を受けるコンポーネント (`core/`, `ui/`, `scripts/`, etc.)
- 再現手順 (最小再現ケース)
- 影響度の見立て (情報漏洩 / RCE / DoS など)
- 提案する修正案 (任意)

### 応答目安

| 重要度 | 初動応答 | 修正目標 |
|---|---|---|
| Critical (RCE、PII 漏洩) | 24 時間以内 | 7 日以内 |
| High (認証回避、Kill-Switch 回避) | 3 日以内 | 14 日以内 |
| Medium | 1 週間以内 | 30 日以内 |
| Low | 2 週間以内 | 次のリリース |

## Kill-Switch 契約

ai-chan の最重要セキュリティ契約は「**消す権利**」です。
関連する脆弱性は最優先で扱います。

- `core/subject_rights.py` の `purge_subject()` がバイパス可能 → **Critical**
- 削除されたはずのデータが残留 → **Critical**
- 月次ドリル (`scripts/killswitch_drill.sh`) がパスしなくなった → **High**

## 脅威モデル

詳細は `docs/THREAT_MODEL.md` を参照してください。

## 暗号化ポリシー

- ローカルストレージ: SQLite + 暗号化レイヤ (`core/encryption.py`)
- 認証情報: 環境変数のみ。ハードコード禁止 (`.gitleaks.toml` で機械検出)
- 通信: HTTPS 必須。自己署名は開発環境のみ

## 既知の境界

以下は **現時点では対象外** (ドキュメント済みの限界):

- サンドボックス脱出 (OS レベル) — Python 実行権限を持つ攻撃者は対象外
- 物理アクセス — デバイスに物理接触できる攻撃者は対象外
- サプライチェーン (PyPI 依存) — `pip-audit` で日次監査するが、完全防御は不可能
