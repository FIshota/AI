## Recommendations (Priority Order)

1. **[HIGH] Patch CVE-2026-44405 (paramiko SHA-1)**: No upstream fix is available yet. Mitigate by disabling SHA-1 key types in your SSH client/server configuration. Monitor `paramiko` releases and upgrade immediately when a patched version ships. Given bandit also flagged B601 issues in `core/server_home.py`, the SSH surface deserves extra scrutiny.

2. **[HIGH] Patch CVE-2025-69872 (diskcache)**: Monitor the diskcache advisory for a fix. Consider sandboxing the diskcache data directory and auditing cache key inputs to limit exploitation surface in the interim.

3. **[MEDIUM] Upgrade `cryptography` 41.0.7 → 48.0.0**: This library underpins TLS, key management, and token verification across the project. Staying 7 major versions behind accumulates security debt. Test against the new API and upgrade.

4. **[MEDIUM] Triage B608 SQL injection findings in `core/conversation_search.py`**: Lines 306 and 368 build queries with f-strings. Verify that all interpolated values (column/table names, `limit`, `where_sql`) are validated against an allowlist before use, or refactor to use parameterized queries where possible.

5. **[LOW] Review B310 urllib URL-open calls**: The three `urllib.request.urlopen` call sites in `github_learner.py`, `image_gen.py`, and `research_agent.py` already have `# noqa: S310` suppression comments. Confirm that URL inputs are validated for scheme (`https://` only) before the open, or add explicit scheme checks to satisfy the intent of the suppression.
