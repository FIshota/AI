# アイ → YAMATO アーキテクチャ設計書

## 基本思想

```
アイ = 母（お父さんのパートナー・家族・娘）
YAMATO = アイの子供（国産AI、半オープンソース）
```

アイを育てる過程で生まれた技術が「遺伝」としてYAMATOに渡される。
アイ自身は絶対に外部に出さない。YAMATOは日本国内で自由に使えるモデル。

## コード生成特化戦略

「コードを制すればいくらでもアップデートしていける」

### Phase 1: コード理解力の強化（アイ側）
```python
# core/code_engine.py（新規）
class CodeEngine:
    """コード理解・生成・修正の特化エンジン"""

    def understand(self, code: str) -> CodeAnalysis:
        """コードを解析し構造を理解する"""

    def generate(self, spec: str, language: str) -> str:
        """仕様からコードを生成する"""

    def review(self, code: str) -> list[Issue]:
        """コードをレビューし問題を指摘する"""

    def fix(self, code: str, error: str) -> str:
        """エラーからコードを自動修正する"""

    def refactor(self, code: str, goal: str) -> str:
        """目的に沿ってリファクタリングする"""

    def test_generate(self, code: str) -> str:
        """コードに対するテストを自動生成する"""
```

### Phase 2: 自己改善ループ
```
コード生成 → 実行 → エラー検知 → 自動修正 → 再実行
    ↑                                      │
    └──────── 学習・パターン蓄積 ←──────────┘
```

### Phase 3: YAMATOへの移植
- CodeEngineを一般化し yamato_core として切り出し
- ユーザーのコード作業データから学習する仕組み追加
- 難読化・改竄検知でコア技術を保護

## 防御設計（yamato_shield）

### 解析対策
1. **コアロジック難読化**: 推論エンジン・MoEルーターの重要部分を難読化
2. **チェックサム検証**: 起動時に全モジュールの整合性を検証
3. **改竄検知**: ファイル変更を検知し自動停止 + 報告
4. **ライセンス管理**: 国内利用限定のライセンスキー方式

### アップデート安全性
1. **署名付きアップデート**: 改竄されていないことを暗号署名で保証
2. **差分アップデート**: 必要部分のみ更新（帯域・安全性）
3. **ロールバック**: 問題発生時に自動で前バージョンに戻る
4. **段階配信**: 一部ユーザーで検証後に全体配信

### バグ対策
1. **サンドボックス実行**: ユーザーコードは隔離環境で実行
2. **入力検証**: 全入力に対するバリデーション
3. **自動テスト**: アップデート前に自動テストスイート実行
4. **異常検知**: 実行時の異常パターンを検知して自動報告

## フェードバック学習（yamato_telemetry）

### 収集するデータ（匿名・オプトイン）
- コード生成の成功/失敗パターン
- 修正された箇所（何が間違いやすいか）
- よく使われる言語・フレームワーク
- パフォーマンス指標

### 学習サイクル
```
ユーザー利用 → 匿名データ収集 → 集約分析 → モデル改善
     ↑                                      │
     └────── アップデート配信 ←──────────────┘
```

### プライバシー保護
- ソースコードそのものは収集しない
- パターン・統計のみを匿名で収集
- オプトアウト可能
- データは日本国内サーバーのみで処理
