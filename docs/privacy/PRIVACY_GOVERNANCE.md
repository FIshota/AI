# ai-chan Privacy Governance

**Status**: Phase 0.1
**Last Updated**: 2026-04-23

> 家族間の約束として、プライバシーに関わる意思決定をどう行うかを記録します。
> 法務目的ではなく、運用上の責任分担と更新手順を明確にするためのメモです。

## 役割分担

| 役割 | 責務 |
|---|---|
| メンテナ | `docs/PRIVACY.md` の更新 / Kill-Switch の動作保証 |
| 家族 | データカテゴリ追加時の合意 / 未成年・ペット・故人の記録方針 |
| ai-chan 本体 | 外部通信時の事前通知 / 監査ログ記録 |

## 更新手順

1. 新しいデータカテゴリを追加する前に、`docs/PRIVACY.md` の表を先に更新する。
2. `scripts/privacy_lint.py` を実行し、必須見出しとカテゴリ整合性を確認する。
3. 実装変更は PR に `privacy-impact` ラベルを付ける。
4. 更新履歴セクションに日付と差分要約を追記する。

## 必須見出し

`docs/PRIVACY.md` は以下の見出しを必ず含む。`privacy_lint.py` が検証する。

- `## 🔑 3 行まとめ`
- `## 📦 データカテゴリ一覧`
- `## 🔴 Kill-Switch`
- `## 🌐 外部通信`
- `## 🧒 未成年`
- `## 📜 更新履歴`

## データカテゴリ整合性

`core/` 以下で新しい永続化モジュールを追加した場合、対応カテゴリを
`docs/PRIVACY.md` に記載する必要がある。`privacy_lint.py` が
既知モジュールリストとの差分を検出する。

## 監査

- 月次: `privacy_lint.py` を CI 上で実行。
- 半期: 家族会議でカテゴリ表をレビュー。
