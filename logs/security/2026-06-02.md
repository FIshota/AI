## Recommendations (Priority Order)

1. **[HIGH] Monitor CVE-2026-44405 (paramiko SHA-1 downgrade)** — No upstream fix available yet. Consider pinning `paramiko<4.0` and adding an SSH host-key algorithm enforcement config (`preferred_algorithms`) to disable SHA-1 in `core/server_home.py`. Watch GHSA-r374-rxx8-8654 for a patched release.

2. **[HIGH] Evaluate diskcache CVE-2025-69872 mitigation completeness** — The existing `_secure_cache_dir()` mitigation is noted in `requirements.txt`. Verify that no attacker-controlled data reaches the cache deserialization path. Consider migrating to a non-pickle backend (e.g., JSON/msgpack) for cache values that originate from external sources.

3. **[MEDIUM] Review SQL construction in `core/conversation_search.py:306,368`** — Column/table names are assembled via f-strings (B608). While these appear to come from internal schema constants (not user input), add an allowlist validation guard to be explicit about the invariant and future-proof the code.

4. **[MEDIUM] Validate URL schemes before `urlopen` calls** — `core/github_learner.py:180`, `core/image_gen.py:156`, `core/research_agent.py:198` use `urllib.request.urlopen`. Add explicit scheme validation (`assert url.startswith(('https://', 'http://'))`) before each call, or use `requests` with `allow_redirects=False` as a safer alternative.

5. **[LOW] Update `pip` to 26.x and `setuptools` to 82.x** — Run `pip install --upgrade pip setuptools` in the project virtualenv. These are build-time tools with no runtime security risk here, but keeping them current avoids known resolver bugs.
