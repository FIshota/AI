## Recommendations (Priority Order)

1. **[P1] Monitor CVE-2026-44405 (paramiko) and CVE-2025-69872 (diskcache).** No upstream fix yet. Subscribe to GitHub release notifications for both packages. If a PoC exploit is published, consider temporarily replacing paramiko with an alternative SSH client or restricting `server_home.py` to trusted network paths.

2. **[P2] Upgrade `cryptography` from 41 → 49.** This is a security-critical library spanning 8 major versions behind. Breaking changes are possible but this is the highest-value outdated upgrade. Test `core/server_home.py` (paramiko) and any TLS-using paths after upgrade.

3. **[P3] Review B601 findings in `core/server_home.py`.** Lines 265/241 pass externally-originated `cmd` to `exec_command`. Verify the call at line 265 sanitizes input before the SSH command is constructed. The static commands (uptime, df, free, docker ps) at lines 298–312 are safe and can be suppressed with `# nosec B601`.

4. **[P4] Harden B608 SQL paths in `core/conversation_search.py`.** The f-string table/column interpolation at lines 306 and 368 uses values that should be validated against an allowlist before use. Add explicit column-name validation to eliminate the SQL injection surface.

5. **[P5] Add `# nosec B310` suppressions to `urlopen` calls** in `github_learner.py`, `image_gen.py`, and `research_agent.py` if URLs are already validated upstream — or add explicit scheme validation (`assert url.startswith(('https://', 'http://'))`) before each call to close the path-traversal vector.
