## 🚨 Critical / High Findings

### HIGH — CVE-2025-69872 · diskcache 5.6.3
- **GHSA**: GHSA-w8v5-vhqr-4h9v
- **説明**: DiskCache がデフォルトで Python pickle を使用してシリアライズ。キャッシュディレクトリへの書き込みアクセスを持つ攻撃者が任意コード実行可能。
- **修正バージョン**: 未公開 (no upstream fix as of 2026-05-23)
- **緩和策**: `requirements.txt` コメント記載通り `_secure_cache_dir()` で mitigated 済み。引き続きキャッシュディレクトリのパーミッション管理を徹底すること。
- **推奨アクション**: upstream の fix をウォッチ。内部実装で `DiskCache(directory, disk=JsonDisk)` 等 pickle 非使用シリアライザへの移行を検討。

## Recommendations (優先度順)

1. **[HIGH] diskcache pickle RCE (CVE-2025-69872) の恒久対策**  
   現行の `_secure_cache_dir()` 緩和に加え、`diskcache.JSONDisk` 等 pickle 非使用バックエンドへの移行を次スプリントで計画。upstream リリースを `pip-audit` で継続監視。

2. **[HIGH] paramiko CVE-2026-44405 修正待ちウォッチ**  
   GHSA-r374-rxx8-8654 の修正リリースを GitHub Advisory で購読。`requirements.txt` の `paramiko<4.0` 上限を修正版で更新。SSH接続時は ed25519/ecdsa-sha2-nistp256 キーを優先設定。

3. **[MEDIUM] conversation_search.py SQL インジェクション対策**  
   `core/conversation_search.py:306,368` の f-string SQL を `?` プレースホルダーに書き換え。カラム名/テーブル名は allowlist で検証済みなら `# nosec B608` を根拠付きで追加。

4. **[MEDIUM] urllib.urlopen スキーム制限**  
   `github_learner.py`, `image_gen.py`, `research_agent.py` の `urlopen` 呼び出しに対し、`http://` / `https://` 以外のスキームを明示的にブロックするバリデーションを追加。

5. **[LOW] cryptography ライブラリの段階的メジャー更新計画**  
   installed 版は既に 48.0.0 で最新だが、`requirements.txt` の下限 `>=46.0.7` を `>=48.0.0` へ引き上げ、古い環境での誤インストールを防ぐ。
