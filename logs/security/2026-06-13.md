## Recommendations (Priority Order)

1. **[CRITICAL — Immediate] Mitigate `torch` CVE-2025-3000**: Until a patched release is available, restrict use of `torch.jit.script` on untrusted inputs; monitor PyTorch security advisories. Consider pinning and monitoring upstream commits (fix commit `a448945`-style). Subscribe to GHSA-rrmf-rvhw-rf47.

2. **[HIGH — This Sprint] Harden `diskcache` against pickle RCE (CVE-2025-69872)**: Set strict filesystem permissions on the cache directory (0700, owned by the service user). Evaluate switching to a non-pickle serializer (`diskcache.JSONDisk`) or replacing `diskcache` with a Redis/memcached backend. Monitor GHSA-w8v5-vhqr-4h9v for a patched release.

3. **[MEDIUM — This Sprint] Update `paramiko` SHA-1 policy (CVE-2026-44405)**: Configure `paramiko` to explicitly disable SHA-1 RSA key exchanges. Use `transport.get_security_options().key_types` to remove `ssh-rsa`. Monitor GHSA-r374-rxx8-8654 for upstream patch.

4. **[MEDIUM — Next Sprint] Review Bandit B601 (paramiko shell injection) in `core/server_home.py`**: `exec_command` calls at lines 241, 265, 298, 302, 306, 312 use string arguments. Audit that all `cmd` values are allowlisted or constructed from constants only; add input validation guards if any path accepts user-controlled data.

5. **[MEDIUM — Backlog] Audit B608 SQL patterns in `core/conversation_search.py`**: Lines 306 and 368 construct SQL via f-strings. Verify that column/table identifiers come only from internal allowlists (not user input). Consider using parameterized identifiers or a schema validation layer.
