## ⚠ CVE Findings (Medium — pip-audit)

### CVE-2026-44405 — paramiko 3.5.1
- **GHSA:** GHSA-r374-rxx8-8654
- **Fix Versions:** なし（未公開）
- **説明:** Paramiko through 4.0.0 において `rsakey.py` が SHA-1 アルゴリズムを許可している。SSH 鍵交換の弱体化につながる可能性がある。
- **推奨:** upstream パッチを監視。回避策として SSH 設定で SHA-1 を明示的に無効化する。

### CVE-2025-69872 — diskcache 5.6.3
- **GHSA:** GHSA-w8v5-vhqr-4h9v
- **Fix Versions:** なし（未公開）
- **説明:** `python-diskcache` はデフォルトで Python pickle によるシリアライズを使用。キャッシュディレクトリへの書き込みアクセスを持つ攻撃者が、読み込み時に任意コード実行 (RCE) を行える。
- **推奨:** キャッシュディレクトリのパーミッションを最小権限に設定。信頼できない入力をキャッシュしない。upstream を監視。

## Recommendations (優先度順)

1. **[HIGH] cryptography を 41.0.7 → 49.0.0 へ更新**
   `pip install --upgrade cryptography` — 暗号ライブラリのメジャー更新はセキュリティパッチを多数含む。最優先。

2. **[MEDIUM] diskcache CVE-2025-69872 の緩和**
   キャッシュディレクトリ (`~/.diskcache/` 等) のパーミッションを `700` に設定し、他ユーザーからの書き込みを防止。upstream の修正パッチが公開され次第アップデート。

3. **[MEDIUM] paramiko CVE-2026-44405 の監視**
   SSH 接続設定でサーバーの `HostKeyAlgorithms` から SHA-1 ベース方式を除外する。upstream パッチを監視。

4. **[LOW] core/server_home.py:265 の `cmd` 変数サニタイズ確認**
   `exec_command(cmd)` に渡す `cmd` が外部入力を含まないことを確認。含む場合はホワイトリスト方式に変更。

5. **[LOW] core/conversation_search.py の SQL クエリ構築見直し**
   現時点では内部変数のみのため低リスクだが、将来の拡張に備えてパラメータ化クエリへの移行を検討。
