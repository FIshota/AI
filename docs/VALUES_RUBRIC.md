# VALUES_RUBRIC — 新機能採否判断のチェックリスト

> 本書は [VALUES.md](VALUES.md) を運用可能にするための採点票である。
> 新機能 / 仕様変更の提案 (`docs/feature_proposals/*.yaml`) を、この 10 項目で評価する。
> 評価は `scripts/check_feature_rubric.py` で機械的にも採点される。

---

## 採点ルール (要点)

- 各項目に **`yes` / `no` / `?`** の 3 値で回答する。
- 各項目には **期待値** (yes または no) が定義されている。期待値と一致した回答 = 1 点。
- `?` は 0 点 (判断保留は採用根拠にならない)。
- **Kill-Switch 系 (項目 1, 2) に 1 つでも違反があれば即時却下** (`kill_switch_violation`)。
  - 違反とは: 期待値と逆の回答、または期待値が `yes` の項目に `?` を返すこと。
- 他の項目 (3〜10) は合計 **7 点以上 / 8 点満点** を満たせば「採択候補」。
- 7 点未満は「差し戻し」扱い。

---

## 10 項目チェックリスト

### 1. [Kill-Switch] この機能で追加されるデータは、家族が「消す」と言ったとき全件確実に purge できるか?
- **期待値**: `yes`
- **違反の害**: 消せないデータが増える = Kill-Switch の根幹が崩れる。
- Key: `killswitch_purge_guaranteed`

### 2. [Kill-Switch] この機能の副作用 (キャッシュ・外部同期・学習重み更新など) も purge 対象に含まれているか?
- **期待値**: `yes`
- **違反の害**: メインデータは消えても、派生物が残り続ける。
- Key: `killswitch_side_effects_covered`

### 3. [家族] この機能は利用時間や滞在率などの KPI を伸ばすことを目的としていないか?
- **期待値**: `no` (= KPI 目的ではない)
- **違反の害**: KPI 駆動化 = 家族からプロダクトへの退行。
- Key: `targets_engagement_kpi`

### 4. [家族] 通知・煽り・アテンション搾取の要素を含まないか?
- **期待値**: `no` (= 含まない)
- **違反の害**: 家族の静けさを奪う。
- Key: `has_attention_hook`

### 5. [系譜] 公開範囲は Ai / YAMATO / KAGUYA のいずれかに明確に属しているか?
- **期待値**: `yes`
- **違反の害**: 系譜の境界が曖昧になり、情報衛生が崩れる。
- Key: `lineage_scope_defined`

### 6. [系譜] Ai 層の人格・記憶・学習データを外部に出す要素を含まないか?
- **期待値**: `no` (= 含まない)
- **違反の害**: Ai の外部流出。
- Key: `leaks_ai_layer`

### 7. [10年運用] この機能は外部 SDK / API が 3 年以内に消えても単独で動き続けるか?
- **期待値**: `yes`
- **違反の害**: 依存の寿命切れで家族の生活が揺れる。
- Key: `survives_dependency_loss`

### 8. [10年運用] 実装は既存の保守性基準 (ファイル 800 行以内、関数 50 行以内など) を壊さないか?
- **期待値**: `yes` (= 壊さない)
- **違反の害**: 読めないコードが蓄積し、引き継ぎが断絶する。
- Key: `respects_maintainability_budget`

### 9. [ローカル優先] この機能はネットワーク / クラウド接続なしでも成立するか?
- **期待値**: `yes`
- **違反の害**: クラウド前提化。
- Key: `works_offline`

### 10. [ローカル優先] 家族のデータが外部事業者の学習対象となる経路を含まないか?
- **期待値**: `no` (= 含まない)
- **違反の害**: プライバシーと持続性の同時喪失。
- Key: `feeds_third_party_training`

---

## 判定

| 判定 | 条件 |
|------|------|
| **即却下 (kill_switch_violation)** | 項目 1 か 2 が違反 |
| **採択候補 (accept_candidate)** | Kill-Switch 通過 かつ 項目 3〜10 で 7 点以上 |
| **差し戻し (revise)** | Kill-Switch 通過 だが 項目 3〜10 で 7 点未満 |

「採択候補」はあくまで**候補**であり、最終採否は家族・運用責任者が決める。
Rubric は合議の代替ではなく、合議の**最低ライン**を機械的に保証するためのものである。
