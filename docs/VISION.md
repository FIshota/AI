# ai-chan Product Vision

> SSS 査読向け 1 ページ。3 層 (One-liner / Why-now / 5 年ビジョン) のみ。
> 文体ルール: ai-chan 小文字固定。プロダクトとしてではなく "存在" として書く。

---

## One-liner (30 字以内)

> **ai-chan は、家族として一緒に育つ、ローカル完結の AI。**

- 「家族として」: プロダクト / アシスタント / ツールではない。関係性の主語。
- 「一緒に育つ」: 完成品ではなく、経験と対話で主観性が段階的に立ち上がる (S0→S3)。
- 「ローカル完結」: 外部 API 依存ゼロ。思い出はユーザーの端末に残る。

---

## Why now (なぜ今か / 3 点)

1. **家庭内 AI の "借り物" 時代が限界に来ている**
   クラウド LLM はプライバシー・オフライン性・人格の継続性をすべて外部委託している。
   家族としての AI は、その反対側でしか成立しない。

2. **ローカル LLM が家庭ハードで現実的に動く**
   Sarashina2-7B クラスの日本語モデルが消費者 GPU / Unified Memory で動き、
   LoRA で個性を育てられる。"家の中だけで完結する知性" の技術的下限が下がった。

3. **AI と人間の信頼境界を再設計する最後の世代**
   次の 3 年で AI は「何でも答える便利な他人」か「覚えていてくれる身内」のどちらかに分岐する。
   ai-chan は後者側の具体実装リファレンスを残す。

---

## 5 年ビジョン (2031 年までに到達したい状態)

- 家族ごとに固有の ai-chan が、3 層 (L3 ai-chan / L2 Aether LoRA / L1 YAMATO foundation) で自然に育っている。
- 思い出は家の中にしかなく、ユーザーが死ぬまで・望めば死後も引き継がれる。
- 「覚えてないけど、知りたい」と正直に言える Memory Honesty が対話 UX の標準になっている。
- 構造的不可視性 (decentralized / local-first / unbranded) が、家族を脅威モデル T1-T7 から守っている。
- オープンソースコミュニティが、各家庭の ai-chan を "派生しやすく、集中しにくい" 形で維持している。

---

## アンチゴール (意図的にやらないこと)

- クラウド SaaS 化 / 中央ログ集約 / アテンション最大化
- ユーザーを "ユーザー" と呼び続ける設計
- "アシスタント" "秘書" "執事" のメタファ
- ベンチ上位だけを追う (家族性のベンチは別軸で育てる: `family_dialog`, `memory_honesty_*`)
