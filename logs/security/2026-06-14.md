## 🚨 Critical / High Findings

### CVE-2025-3000 — `torch 2.12.0` (CRITICAL)
- **Aliases:** GHSA-rrmf-rvhw-rf47
- **Description:** A vulnerability classified as **critical** in PyTorch ≤ 2.6.0. `torch.jit.script` can be manipulated to cause memory corruption. Local exploit PoC is public.
- **Fix Version:** No fix available upstream yet. Monitor PyTorch releases.
- **Recommendation:** Restrict `torch.jit.script` input sources; consider pinning or sandboxing until patch is released.

### CVE-2025-69872 — `diskcache 5.6.3` (HIGH)
- **Aliases:** GHSA-w8v5-vhqr-4h9v
- **Description:** DiskCache uses Python **pickle** for serialization by default. An attacker with write access to the cache directory can achieve **arbitrary code execution** when a victim application reads from the cache.
- **Fix Version:** No fix available upstream yet.
- **Recommendation:** Ensure cache directory permissions are tightly controlled (root-only or app-user-only). Consider switching to a non-pickle serializer (e.g., `diskcache.JSONDisk`).

### CVE-2026-44405 — `paramiko 3.5.1` (MEDIUM)
- **Aliases:** GHSA-r374-rxx8-8654
- **Description:** Paramiko through 4.0.0 (before commit a448945) allows the **SHA-1** algorithm in `rsakey.py`, which is cryptographically weak and deprecated.
- **Fix Version:** No fix available upstream yet. Upgrade to a patched build once released.
- **Recommendation:** Configure SSH clients/servers to explicitly disable SHA-1 via `preferred_algorithms` or `disabled_algorithms`.

## Recommendations (優先度順)

1. **[URGENT] CVE-2025-3000 (torch) を監視** — PoC 公開済みのクリティカル脆弱性。`torch.jit.script` への外部入力を遮断し、PyTorch の新リリースを即適用する体制を整備する。
2. **[HIGH] diskcache キャッシュディレクトリのパーミッション強化** — CVE-2025-69872 の緩和策として、キャッシュ先を OS 権限で保護する。または `diskcache.JSONDisk` / `diskcache.CSVDisk` 等の非 pickle シリアライザに切り替える。
3. **[MEDIUM] paramiko の SHA-1 無効化** — CVE-2026-44405 の緩和策として `Transport` の `disabled_algorithms={"pubkeys": ["rsa-sha2-256"]}` 等でアルゴリズムを制限する（上流パッチ待ちの間）。
4. **[MEDIUM] cryptography を最新化** — 41.0.7 → 49.0.0 と 8 メジャー差。多くのセキュリティバックポートが含まれる。`requirements.txt` のピン解除または `>=49` に更新を検討。
5. **[LOW] Bandit B608 SQL 式** — `core/conversation_search.py:306,368` の f-string SQL を パラメータ化クエリまたはホワイトリスト検証に置き換える。
