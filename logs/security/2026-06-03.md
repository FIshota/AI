## Recommendations (優先度順)

1. **[P0 — CRITICAL] diskcache pickle RCE (CVE-2025-69872)**
   - 即時: キャッシュディレクトリのパーミッションを `chmod 700` + サービスアカウント所有に制限
   - 短期: upstream の fix がリリースされ次第 `diskcache>=5.6.4`（または代替バージョン）へ更新
   - 中期: pickle 以外のシリアライザ（`msgpack`/`json`）への切替を検討（`DiskCache(serializer=…)`）

2. **[P1 — HIGH] paramiko SHA-1 (CVE-2026-44405)**
   - 即時: `SSHClient.connect()` 呼び出し全箇所に `disabled_algorithms={"keys": ["rsa-sha1"]}` を追加
   - 短期: 公式 fix リリース後に `paramiko>=4.x` へ更新（API 差分検証が必要）

3. **[P2 — MEDIUM] Bandit B608 SQL injection (conversation_search.py:306, :368)**
   - `core/conversation_search.py` の f-string クエリを SQLite `?` プレースホルダーまたはホワイトリスト検証に置き換え
   - テーブル名・カラム名は定数テーブルで検証する（動的 DDL は `nosec` 明示 + コメント必須）

4. **[P2 — MEDIUM] Bandit B310 urllib urlopen (3 箇所)**
   - URL スキームを `https://` のみに制限するバリデーションを追加
   - 既存 `# noqa: S310` コメントは意図的だが、入力がユーザー制御可能でないことを確認

5. **[P3] 依存関係のメジャー更新評価**
   - `cryptography` システム層を 48.x に揃える（pip 環境は既に対応済み）
   - `setuptools 82.x`、`packaging 26.x` のリリースノートを確認し requirements.txt の上限を解除
