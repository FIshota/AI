## Recommendations

### Priority 1 — Mitigate diskcache RCE (CVE-2025-69872)
Switch to non-pickle serialization immediately:
```python
from diskcache import Cache, JSONDisk
cache = Cache('/path/to/cache', disk=JSONDisk)
```
Also restrict cache directory permissions: `chmod 700 /path/to/cache`. Monitor upstream for a patched release.

### Priority 2 — Mitigate paramiko SHA-1 + exec_command variable (CVE-2026-44405 + B601)
These two compound each other. At `core/server_home.py:265`, validate `cmd` against an allowlist before passing to `exec_command`. Example:
```python
ALLOWED_CMDS = {"status", "restart", ...}
if cmd not in ALLOWED_CMDS:
    raise ValueError(f"Command not permitted: {cmd}")
```
For SHA-1: explicitly disable with `disabled_algorithms={'keys': ['rsa-sha2-256', 'rsa-sha2-512']}` only if SHA-1 RSA auth is not needed. Monitor paramiko for patch post-commit `a448945`.

### Priority 3 — Review SQL injection surface (B608, conversation_search.py)
Verify that table/column names and `where_sql`/`join` fragments in `core/conversation_search.py:306,368` never originate from user-controlled input. If they do, enforce an explicit allowlist. If column/table names are purely internal constants, add a `# nosec B608 — column names are internal constants` comment.

### Priority 4 — Update cryptography package
The `cryptography` gap (41.0.7 → 48.0.0) spans 7 major versions. Confirm the project venv is fully isolated from the system package and add `cryptography>=48.0.0` to `requirements.txt` if not already present.

### Priority 5 — Review urlopen URL validation (B310)
The three `# noqa: S310` suppressions at `github_learner.py:180`, `image_gen.py:156`, and `research_agent.py:198` are accepted risks. Confirm that URLs passed to `urlopen` are either hardcoded or validated to `https://` scheme only before the call.
