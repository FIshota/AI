# アイちゃん アップデートプラン 100選
## 各部門視点 × 現在の開発環境で実行可能な緻密な実装計画

> **対象環境**: Intel Mac / Python 3.9 / Qwen2.5-3B-4bit / MLX非フル対応
> **優先度**: ★★★高 / ★★中 / ★低
> **難易度**: 🟢簡単（1日以内） / 🟡中程度（1週間） / 🔴高（1ヶ月以上）

## 🔧 エンジニア視点（20プラン）

### E-01 ★★★ 🟢 CodeEngine → 会話パイプライン接続
**概要**: `code_engine.py`が孤立しており、会話フローから呼べない  
**実装**:
- `ai_chan.py`の`_route_command()`に`/code`, `/review`, `/fix`コマンドを追加
- `CodeEngine.analyze()` / `review()` / `suggest_fix()`を会話レスポンスに変換するアダプタを追加
- ファイルパスはコマンド引数で渡す（例: `/code analyze ~/project/main.py`）
**期待効果**: コード解析・レビューが会話から直接呼び出せるようになる

---

### E-02 ★★★ 🟢 self_will.pyのアクション拡充
**概要**: 現在「learn_topic」1種類しかない自意志アクションを5種以上に拡張  
**実装**:
```python
# self_will.py に追加するアクション
ACTIONS = {
    "review_own_code":    "自分のソースコードを定期レビュー",
    "organize_memory":    "重複・古い記憶を圧縮・整理",
    "check_health":       "システムヘルスチェック実行",
    "practice_dialogue":  "過去会話から対話パターンを練習",
    "web_research":       "興味トピックをWeb検索して知識補充",
}
```
**期待効果**: アイちゃんが自律的に多様な行動を取れるようになる

---

### E-03 ★★★ 🟢 main.py Python互換性修正
**概要**: `str | None`型ヒント構文がPython 3.9で動かない（3.10以上が必要）  
**実装**: `from __future__ import annotations`を各ファイル先頭に追加、または`Optional[str]`に統一  
**対象ファイル**: main.py, core/*.py（`|`演算子を使っている全箇所）  
**期待効果**: 任意のPython 3.9環境で`python main.py`が動くようになる

---

### E-04 ★★★ 🟡 非同期タスクキュー実装
**概要**: Web取得・学習・定期処理がメインスレッドをブロックする問題  
**実装**:
```python
# core/task_queue.py
import asyncio, queue, threading
class BackgroundQueue:
    """低優先度タスクをキューイングして順次実行"""
    def submit(self, coro, priority=5): ...
    def drain_idle(self): ...  # 会話の合間に処理
```
**期待効果**: 会話中に重い処理が走っても応答が止まらなくなる

---

### E-05 ★★ 🟢 応答ストリーミング改善
**概要**: LLM出力が全部出てから表示されるためラグが大きい  
**実装**: `llm.py`の`generate()`でMLXのストリーミングAPIを使い、`ui/cli.py`でチャンクごとに表示  
**期待効果**: 長い返答でも最初の1文が即座に表示される

---

### E-06 ★★ 🟡 プラグインホットリロード
**概要**: `core/plugins/`のプラグインを再起動なしで読み込み直せるようにする  
**実装**: `plugin_loader.py`に`watchdog`でファイル変更を検知 → `importlib.reload()`で再ロード  
**期待効果**: 開発中にプラグインを追加・変更してもアイちゃんを再起動不要

---

### E-07 ★★★ 🟡 APIサーバーモード
**概要**: `FastAPI`でHTTP APIを公開し、他アプリからアイちゃんを呼べるようにする  
**実装**:
```python
# core/server_ai_env.py を拡張
POST /api/chat       → 会話
POST /api/command    → コマンド実行
GET  /api/status     → ヘルス情報
GET  /api/memory     → 記憶取得
```
**期待効果**: Webアプリ・Discord Bot・Slackからアイちゃんを呼び出せる

---

### E-08 ★★ 🟢 設定ホットリロード
**概要**: `config/settings.json`を書き換えても再起動が必要な問題  
**実装**: `config_watcher.py`を`chrome()`と接続し、変更検知 → `settings`オブジェクトをリフレッシュ  
**期待効果**: 感情パラメータや閾値を実行中に調整できる

---

### E-09 ★★ 🟡 コンテキストウィンドウ最適化
**概要**: 3Bモデルの400トークン上限に引っかかり文章が途切れる問題

### E-10 ★★ 🟢 構造化ログ出力

**期待効果**: ログ解析・デバッグが大幅に速くなる

### E-11 ★★★ 🟡 WebSocketリモート接続
