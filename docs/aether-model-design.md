
### 3.3 Phase 2: Aether v0.5 "Forge"（鍛造）

**目標**: 独自トークナイザー + 深い知識蒸留で「実質オリジナル」へ

#### 独自トークナイザー "Yamato Tokenizer"

```
目的: 日本語に最適化された語彙で、同じ文章をより少ないトークンで表現する

既存:  Qwen 152K語彙 → 日本語「こんにちは、今日はいい天気ですね」 = 8-10 tokens
目標:  Yamato 48K語彙 → 同じ文章 = 5-6 tokens

手法:
1. 大量の日本語テキストから BPE/Unigram を学習
2. あいちゃんの会話で頻出するフレーズを優先的にトークン化
3. 語彙サイズを最適化（大きすぎると埋め込み層が重い）
4. 48K-64K語彙が3Bモデルの最適点
```

なぜ48-64Kなのか:
- 語彙が大きい = 埋め込み層が大きい = モデルサイズ増加
- 語彙が小さい = トークン数が増える = 推論が遅い
- 3Bモデルの場合、48-64Kが計算効率と表現力のスイートスポット

#### 知識蒸留パイプライン

```
教師モデル（大型）               生徒モデル（Aether）
┌─────────────────┐             ┌──────────────────┐
│ Claude / GPT-4  │──生成──→    │                  │
│ Qwen 72B        │  合成データ  │  Qwen 2.5 3B     │
│ Gemma 27B       │──蒸留──→    │  + YamatoTokenizer│
│                 │  知識        │  + LoRA           │
└─────────────────┘             └──────────────────┘

蒸留の種類:
1. 応答蒸留: 大型モデルの応答を学習データとして使う（M2 Proで可能）
2. ロジット蒸留: 大型モデルの確率分布を模倣（GPU必要）
3. 特徴蒸留: 中間層の表現を模倣（GPU必要）

Phase 2ではまず「応答蒸留」に集中。APIで大型モデルを呼び、
その応答を訓練データとして使う。これならM2 Proで完結。
```
