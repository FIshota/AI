## 🚨 Critical / High Findings

### CVE-2025-69872 — diskcache 5.6.3 (HIGH)
- **Package**: diskcache 5.6.3
- **GHSA**: GHSA-w8v5-vhqr-4h9v
- **Fix version**: None available yet (upstream not patched)
- **Description**: DiskCache through 5.6.3 uses Python pickle for serialization by default. An attacker with write access to the cache directory can achieve **arbitrary code execution** when a victim application reads from the cache.
- **Impact**: If the cache directory (`~/.cache/` or similar) is accessible to a local attacker or via path traversal, malicious pickle payloads could execute arbitrary code.
- **Mitigation**: Restrict cache directory permissions (`chmod 700`); consider disabling pickle serialization or migrating to a safer serializer.

## ⚠ Medium Findings (pip-audit)

### CVE-2026-44405 — paramiko 3.5.1 (MEDIUM)
- **Package**: paramiko 3.5.1
- **GHSA**: GHSA-r374-rxx8-8654
- **Fix version**: None listed (apply when available, target `>= a448945` commit)
- **Description**: Paramiko through 4.0.0 before commit a448945 allows the **SHA-1 algorithm** in RSA key operations, enabling weak cryptographic primitives.
- **Impact**: SSH connections using RSA keys may fall back to SHA-1 signatures, reducing cryptographic strength.
- **Mitigation**: Explicitly disable SHA-1 in paramiko client/server configuration; prefer ED25519 or ECDSA keys.
