## Recommendations (優先度順)

1. **[HIGH] CVE-2026-44405 (paramiko) — Monitor for patch**: paramiko の SHA-1 RSA 鍵処理の脆弱性。現時点で upstream に fix バージョンなし。`core/server_home.py` の SSH 接続で使用中のため、upstream リポジトリを監視し patch リリース次第即時更新する。暫定対策として RSA 鍵を Ed25519 へ移行を検討。

2. **[HIGH] CVE-2025-69872 (diskcache) — Monitor for patch**: diskcache 5.6.3 の CVE。fix バージョン未リリース。キャッシュ機構への不正アクセスリスクがあるため、upstream を監視しつつキャッシュへのネットワーク露出を最小化する。

3. **[MEDIUM] B608 SQL Injection リスク (`core/conversation_search.py`)**: f-string での SQL 構築は bandit が潜在的 SQL インジェクションと判定。テーブル名・カラム名がユーザー入力に由来しないか確認し、ホワイトリスト検証を追加するか `# nosec B608` で意図的抑制を明示する。

4. **[MEDIUM] B601 Paramiko exec_command (`core/server_home.py`)**: `exec_command` に渡すコマンド文字列が外部入力を含む場合はシェルインジェクションリスク。現状は静的文字列のみのため低リスクだが、将来の改修時に動的コマンド組み立てを禁止するコードレビュールールを設ける。

5. **[LOW] cryptography 環境不一致**: システム環境の cryptography 41.0.7 と requirements.txt の 48.0.0 が乖離。仮想環境またはコンテナで実行することを確認し、`pip install -r requirements.txt` を CI で強制する。
