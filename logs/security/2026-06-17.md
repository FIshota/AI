## Recommendations（優先度順）

1. **🔴 [優先1] paramiko CVE-2026-44405 対応**  
   upstream パッチがリリースされ次第、即時アップデート。  
   それまでは `core/server_home.py` の exec_command に渡す入力の厳格なバリデーションを強化し、  
   ホワイトリスト方式でコマンドを制限することを推奨。

2. **🔴 [優先2] cryptography を 46.x → 49.x へ更新**  
   `requirements.txt` は `>=46.0.7` だが実行環境に 41.x が入っている可能性あり。  
   セキュリティライブラリの最新化は高優先度。  
   `pip install "cryptography>=49.0.0"` で更新後、`pip-audit` で再検証。

3. **🟡 [優先3] diskcache CVE-2025-69872 モニタリング**  
   `_secure_cache_dir()` による mitigated だが、upstream パッチ動向を継続監視。  
   patch リリース時は即時適用。

4. **🟡 [優先4] Bandit B608 (SQL injection) レビュー**  
   `core/conversation_search.py:306` および `:368` の f-string SQL クエリを  
   パラメータバインディング方式に変更することを検討。  
   現在 `#nosec` が適用されていないため、意図的か確認が必要。

5. **🟢 [優先5] torch CVE-2025-3000 評価**  
   torch は `requirements.txt` 上 optional 扱い（コメントアウト推奨）。  
   本番環境に torch が入っていなければリスク低。  
   optional 依存として明示し、`pip-audit` の除外対象に追加することを検討。
