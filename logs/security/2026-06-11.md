## Recommendations（優先度順）

1. **[P1 — 即時対応] `torch` CVE-2025-3000 を追跡する**
   公式修正がリリースされ次第、速やかに `requirements.txt` を更新すること。
   それまでは `torch.jit.script` への外部入力を絶対に渡さない実装ガードを追加する。

2. **[P1 — 即時対応] `paramiko` CVE-2026-44405 — SHA-1 無効化**
   修正版リリースまでの緩和策として、`paramiko` の `RSAKey` / Transport 設定で
   `disabled_algorithms={"keys": ["rsa-sha2-256"], "pubkeys": ["rsa-sha2-256"]}` を
   明示設定し SHA-1 ネゴシエーションを強制無効化する。

3. **[P2 — 今週中] `diskcache` CVE-2025-69872 — Pickle 逆シリアル化**
   DiskCache の書き込み権限を最小化し、キャッシュディレクトリを非公開パスに限定する。
   可能であれば `DiskCache(disk=JSONDisk)` で JSON シリアライザに切り替える。

4. **[P2 — 今週中] `cryptography` 41→48 にアップグレード**
   7 メジャーバージョンの乖離はセキュリティパッチを多数含む。
   `pip install -U cryptography` でアップグレード後、テストを実行して確認する。

5. **[P3 — 今月中] Bandit B608 SQL インジェクション対策**
   `core/conversation_search.py:306` および `:368` の f-string クエリを
   プリペアドステートメント (`?` プレースホルダ) に置き換える。
