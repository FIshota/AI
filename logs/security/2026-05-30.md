## 🚨 Critical / High Findings

### CVE-2026-44405 — paramiko 3.5.1
- **Severity:** HIGH
- **Package:** `paramiko` 3.5.1
- **Fix Version:** None available yet (upstream: apply patch [a448945](https://github.com/paramiko/paramiko/commit/a448945))
- **Aliases:** GHSA-r374-rxx8-8654
- **Description:** Paramiko through 4.0.0 (before commit a448945) allows the SHA-1 algorithm in RSA key operations. SHA-1 is cryptographically broken and may allow downgrade or collision attacks in key exchange.
- **Recommendation:** Monitor paramiko upstream; pin to patched commit or apply vendored patch. Consider disabling SHA-1 key types in SSH configuration.

### CVE-2025-69872 — diskcache 5.6.3
- **Severity:** HIGH
- **Package:** `diskcache` 5.6.3
- **Fix Version:** None available yet
- **Aliases:** GHSA-w8v5-vhqr-4h9v
- **Description:** DiskCache through 5.6.3 uses Python `pickle` for serialization by default. An attacker with **write access to the cache directory** can achieve arbitrary code execution (RCE) when a victim application reads from the cache.
- **Recommendation:** Restrict filesystem permissions on the cache directory (mode 0700, owned by service user). Evaluate migrating to a non-pickle serializer if `diskcache` exposes the cache to untrusted paths.
