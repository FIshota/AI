## Recommendations (priority order)

1. **[HIGH] Mitigate CVE-2025-69872 (diskcache pickle RCE)** — Until a patched release is available, restrict filesystem permissions on the cache directory so only the application user can write. Consider switching to a safer serializer (e.g., `diskcache` with `disk=JSONDisk`) or replacing diskcache with a Redis/memcached backend. This is an arbitrary code execution vector if an attacker can write to the cache directory.

2. **[HIGH] Mitigate CVE-2026-44405 (paramiko SHA-1)** — Pin paramiko to commit `a448945` or apply the patch manually to `rsakey.py`. Alternatively, enforce `PubkeyAcceptedAlgorithms -ssh-rsa` on SSH server configs to reject SHA-1 negotiation at the server side. Monitor for an official release.

3. **[MEDIUM] Review Paramiko exec_command usage in `core/server_home.py`** — The `cmd` variable passed to `exec_command` at line 265 must be validated/allowlisted before use. Static commands (lines 241, 298, 302, 306, 312) are low risk as-is but consider using channel-level commands instead of shell strings to eliminate interpreter injection surface.

4. **[MEDIUM] Upgrade `cryptography` (41.0.7 → 48.0.0)** — 7 major versions behind; this library underpins TLS, key management, and SSH throughout the codebase. A major upgrade may include breaking API changes but is strongly recommended for security posture.

5. **[LOW] Audit SQL f-string construction in `core/conversation_search.py`** (lines 306, 368) — Confirm that all interpolated identifiers (`id_c`, `ts_c`, `tbl`, etc.) are derived from an allowlist of known column/table names, not user input. If any value originates from user input, switch to parameterized identifiers or a query builder.
