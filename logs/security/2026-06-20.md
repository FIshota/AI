## 🚨 Critical / High Findings (CVEs)

### CVE-2026-44405 — `paramiko 3.5.1` (HIGH)
- **GHSA:** GHSA-r374-rxx8-8654
- **Fix Version:** No official release fix yet — patch via commit `a448945`
- **Description:** Paramiko through 4.0.0 (before commit a448945) permits the SHA-1 algorithm in `rsakey.py`, enabling downgrade attacks against RSA host-key verification.
- **Impact:** SSH connections using RSA keys may be vulnerable to signature forgery via SHA-1 collision.

### CVE-2025-69872 — `diskcache 5.6.3` (HIGH)
- **GHSA:** GHSA-w8v5-vhqr-4h9v
- **Fix Version:** No official release fix yet
- **Description:** DiskCache (python-diskcache) through 5.6.3 uses Python `pickle` for serialization by default. An attacker with write access to the cache directory can achieve **arbitrary code execution** when a victim application reads from the cache.
- **Impact:** If the cache storage path is attacker-writable, RCE is possible on cache reads.

## Recommendations (優先度順)

1. **[P1 — 即時] diskcache CVE-2025-69872 緩和:** キャッシュディレクトリ権限を `chmod 700` でアプリユーザー専用に制限。公式修正まで `DiskCache(…, disk=diskcache.JSONDisk)` 等 pickle 以外のシリアライザへ移行を検討。27日間未対処のため早急な対応が必要。

2. **[P1 — 即時] cryptography を 49.0.0 へ更新:** `requirements.txt` の `cryptography` が 41.x 系 — 49.x に比べ8メジャーバージョン、多数の脆弱性修正が含まれる。`paramiko` も間接依存するため優先度高。

3. **[P2 — 今週] paramiko CVE-2026-44405 対応:** `paramiko` を commit `a448945` 以降を含むバージョンに更新、または公式修正リリースを待ちピン留め。`core/server_home.py` の SSH 接続が直接影響範囲。

4. **[P3 — 今スプリント] B608 SQL インジェクション対策:** `core/conversation_search.py:306,368` の f-string SQL を識別子ホワイトリスト検証に書き換え。SQLite パラメータバインドは識別子を保護できないため別途対策が必要。

5. **[P3 — 今スプリント] B601 Paramiko `exec_command` 入力検証:** `core/server_home.py:265` の `cmd` 変数渡し箇所が外部入力を含む場合はホワイトリスト検証を追加。静的リテラル (`"uptime -p"` 等) は低リスクだが変数呼び出し箇所は要確認。
