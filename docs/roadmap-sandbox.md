# F7: コード実行サンドボックス（将来実装）

## 概要
アイが自分でPythonコードを書いて安全に実行できる環境。

## 設計方針
- サンドボックス環境（制限付きPython実行）
- ファイルシステムアクセス制限（`data/sandbox/` のみ）
- ネットワークアクセス制限
- 実行時間制限（5秒）
- メモリ制限

## ユースケース
- 「計算して」→ 数式を解く
- 「グラフ描いて」→ matplotlib で可視化
- 「データ分析して」→ pandas で集計
- 「テスト書いて」→ 自分のコードのテストを生成

## 安全設計
```
ユーザーの依頼 or アイの意思
      ↓
コード生成（LLM）
      ↓
安全チェック（禁止パターン検出）
  - import os / subprocess / sys → ブロック
  - open() でサンドボックス外 → ブロック
  - exec() / eval() のネスト → ブロック
  - ネットワーク系 → ブロック
      ↓
サンドボックスで実行（制限付き）
  - RestrictedPython or subprocess with limits
  - timeout=5s, memory limit
      ↓
結果をユーザーに報告
```

## 必要な前提条件
- 信頼の蓄積（成長段階が青年期以上）
- 自己修正システムが安定稼働している
- 自己開発パイプラインで提案経験が十分ある

## 実装難易度: ★★★★★
セキュリティ設計が最も複雑。慎重に進める。
