# 輸出管理 (Export Control) 注意事項 — Ai-chan

> **重要 Disclaimer**: 本書は**法的助言 (legal advice) ではなく、運用上の注意喚起 (operational awareness)** を目的とした開発者向けメモです。実際の輸出判定・該非判定については、**必ず経済産業省・法務専門家・企業の輸出管理部門に確認**してください。記載内容には誤りや時点ズレの可能性があります。

## 1. 関心領域 (Areas of Concern)

Ai-chan には輸出管理上、注意を要する**可能性がある**技術要素が含まれています。

| 要素 | 実装箇所 (例) | 懸念カテゴリ |
|------|--------------|-------------|
| 暗号機能 (AES等対称鍵暗号) | `cryptography` ライブラリ経由、macOS Keychain 連携 | 情報セキュリティ関連貨物・技術 |
| ML モデル (LLM, STT, TTS) | `models/` 配下、YAMATO/hinomoto-model | 先端 AI 技術 (各国で新規規制の可能性) |
| 多言語処理 | 日本語・英語・その他言語間の変換 | 一般に非該当の可能性が高いが、軍事翻訳用途は別 |
| ハッシュ・認証 (HMAC, hashlib) | 整合性検証 | 多くは汎用技術として除外される可能性 |

## 2. 規制フレーム概要 (参考)

| 規制 | 管轄 | 主な対象 | 本プロジェクトとの関係 (推測) |
|------|------|---------|------------------------------|
| 外為法・輸出貿易管理令 | 日本 / 経済産業省 | リスト規制 (別表第一) ・キャッチオール規制 | 暗号項目 (7項・9項) 該当性を要確認 |
| EAR (Export Administration Regulations) | 米国 / BIS | ECCN 5A002, 5D002 (暗号) / 3A/4A (計算機) / AI 関連 | 米国製依存ライブラリ経由で再輸出規制の可能性 |
| Wassenaar Arrangement | 多国間 | 通常兵器・機微汎用品 | 暗号機能が Category 5 Part 2 に関連する可能性 |
| EU Dual-Use Regulation (2021/821) | EU | サイバー監視品目等 | EU 域内配布時に考慮の可能性 |

## 3. リスク評価 (暫定)

以下はあくまで**開発者の自己評価に基づく推測**であり、確定的な該非判定ではありません。

- **個人使用 / 非商用限定**: 個人のローカル環境での研究・学習用途に留まる場合、輸出行為自体が発生しないため、**原則非該当の可能性** [要法務確認]
- **GitHub 等 Public リポジトリ経由の配布**: 多くの司法管轄で「公開されている情報 (publicly available / published)」に対する**オープンソース例外**が存在する**可能性** がある。ただし:
  - 日本の外為法では「公知」となった技術は規制対象外となる場合があるが、暗号ソフトウェアは慎重な判断が必要な**可能性**
  - 米国 EAR では §734.7, §734.3(b)(3) による「publicly available」除外の可能性があるが、事前通知 (TSU notification to BIS) が必要なケースもある**可能性**
- **バイナリ配布 (AiChan.app)**: ソースコードと比べ、規制判断が異なる**可能性** がある
- **禁輸国・制裁対象者への提供**: いかなる公開例外も適用されない**可能性** が高い。OFAC SDN List / 日本の外国ユーザーリスト等の確認が必要

## 4. 運用上の推奨事項 (非拘束)

- [ ] 商用化・法人配布を検討する段階で、**必ず法務レビューを実施**する
- [ ] 新規暗号アルゴリズム追加時は `scripts/check_crypto_surface.py` を実行し差分を記録
- [ ] 禁輸国ユーザーからの明らかなアクセスが検出された場合の対応ポリシーを整備する**可能性** を検討
- [ ] README の Legal セクションに本書へのリンクを維持
- [ ] 依存ライブラリ (特に米国 origin) のライセンス及び輸出管理分類番号 (ECCN) を可能な範囲で記録

## 5. オープンソース公開例外 (参考メモ)

- **日本**: 外為法施行令等により、「公知の技術」「不特定多数への公開」に該当する場合、役務取引許可が不要となる**可能性**
- **米国 EAR**: "publicly available" encryption source code (§742.15(b)) — BIS への通知が必要な**可能性**
- これらは司法管轄ごとに要件・例外が異なり、解釈も流動的である。**該当性の自己判断は避ける**こと
