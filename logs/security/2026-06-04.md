## 🚨 Critical / High Findings (pip-audit)

### HIGH — CVE-2025-69872 · diskcache 5.6.3
- **GHSA**: GHSA-w8v5-vhqr-4h9v
- **Package**: `diskcache` 5.6.3
- **Fix version**: ⚠ No upstream fix released as of scan date
- **Description**: DiskCache through 5.6.3 uses Python `pickle` for serialization by default. An attacker with write access to the cache directory can achieve **arbitrary code execution** when a victim application loads cached data.
- **Impact**: RCE if cache directory is writable by an untrusted party (e.g., shared container storage, writable mounts).
- **Recommendation**: Audit all cache directories' write permissions; consider switching to a safer serializer or upgrading once a fix is released.

### MEDIUM — CVE-2026-44405 · paramiko 3.5.1
- **GHSA**: GHSA-r374-rxx8-8654
- **Package**: `paramiko` 3.5.1
- **Fix version**: ⚠ No upstream fix released as of scan date (fixed in commit `a448945` per advisory, no release yet)
- **Description**: In Paramiko through 4.0.0 (before commit `a448945`), `rsakey.py` allows the SHA-1 algorithm. SHA-1 is cryptographically weak for RSA signatures.
- **Impact**: Downgrade attack risk on SSH connections when SHA-1 is negotiated.
- **Recommendation**: Pin to a patched commit or monitor for new release. Disable SHA-1 in SSH server/client configuration as a mitigation.

2. **[HIGH — Short-term]** Upgrade `cryptography` from 41.0.7 to 48.x. This is a security-critical library that is 7 major versions behind. Run `pip install --upgrade cryptography` and validate no breaking API changes in dependent code.

3. **[MEDIUM — This Sprint]** Audit `core/server_home.py:265` — the dynamic `cmd` variable passed to `paramiko exec_command`. Verify the command is constructed from a strict allowlist and cannot be influenced by user input. This is the only non-hardcoded paramiko call.
