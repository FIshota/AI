## 🚨 Critical / High Findings

### CVE-2025-69872 — diskcache 5.6.3 (HIGH)
- **Package**: `diskcache` 5.6.3
- **GHSA**: GHSA-w8v5-vhqr-4h9v
- **Fix Version**: None available yet
- **Description**: DiskCache uses Python `pickle` for serialization by default. An attacker with write access to the cache directory can achieve **arbitrary code execution** when a victim application reads from the cache.
- **Recommendation**: Audit all cache directories for unexpected write permissions. Consider switching to a non-pickle serializer or applying strict directory ACLs until a fix is released.

## Recommendations (Priority Order)

1. **[HIGH] diskcache pickle RCE (CVE-2025-69872)**: Until a patched release is available, restrict write access to the cache directory to the application user only (`chmod 700`). Evaluate replacing pickle serializer with `json` or `msgpack` in the DiskCache configuration.

2. **[HIGH] Upgrade `cryptography` 41.0.7 → 48.0.0**: This security-critical package is 7 major versions behind. Many CVEs have been patched in between. Upgrade and run tests against the new API.

3. **[MEDIUM] paramiko SHA-1 (CVE-2026-44405)**: Monitor the paramiko GitHub for a patched release. In the interim, avoid RSA key operations over SHA-1 by enforcing `disabled_algorithms` in SSH client configuration.

4. **[MEDIUM] SQL injection surface in `core/conversation_search.py`**: Lines 306 and 368 use f-strings to construct SQL queries with table/column names. Validate these identifiers against an allowlist before interpolation.

5. **[MEDIUM] Paramiko `exec_command` hardening in `core/server_home.py`**: Lines 241, 265, 298, 302, 306, 312 pass commands to `exec_command`. Ensure all commands are hardcoded literals (not user-controlled). Add input validation where `cmd` (line 265) is derived from external input.
