## Recommendations (優先度順)

1. **[HIGH] diskcache キャッシュディレクトリのパーミッション強化**  
   CVE-2025-69872 の upstream fix がないため、cache ディレクトリ (`~/.cache/ai-chan` 等) を `chmod 700` に設定し、ai-chan プロセス以外の書き込みを禁止する。また diskcache の `disk_pickle_protocol` を無効化するか、`DjangoCache`/カスタムシリアライザで pickle を回避することを検討。

2. **[HIGH] `cryptography` パッケージを 48.x へ更新**  
   41.x から 48.x の間に複数の CVE が修正されている可能性が高い。`pip install --upgrade cryptography` を実施し、動作確認後 `requirements.txt` を更新する。

3. **[MEDIUM] `core/conversation_search.py` の SQL インジェクション対策**  
   `:306` および `:368` の f-string による SQL 構築を SQLite のプレースホルダ (`?`) またはパラメータバインディングへ置き換える。テーブル名・カラム名は allowlist で検証する。

4. **[MEDIUM] `core/server_home.py` Paramiko `exec_command` の入力検証**  
   `:265` はユーザー由来の `cmd` を直接渡している可能性がある。コマンドを列挙型 allowlist (enum) で制限し、任意文字列の実行を防ぐ。

5. **[LOW] `urllib.request.urlopen` を `requests` ライブラリへ統一**  
   B310 指摘箇所 (github_learner, image_gen, research_agent) は、URL スキームを `https://` のみに制限するバリデーション (`utils/url_guard.py` が既存) を適用するか、`requests` ライブラリ経由に統一してスキーム制御を明示する。
