## 🚨 Critical / High Findings (CVE)

### CVE-2026-44405 · paramiko 3.5.1 · HIGH
- **Aliases:** GHSA-r374-rxx8-8654
- **Fix versions:** None available (upstream patch commit: `a448945`)
- **Description:** In Paramiko through 4.0.0 before commit a448945, `rsakey.py` permits the SHA-1 signature algorithm, which is cryptographically weak and deprecated. Enables downgrade attacks against SSH connections that rely on RSA key authentication.
- **Scope:** `core/server_home.py` (SSH接続に paramiko を使用)
- **Workaround:** Pin to a post-`a448945` git snapshot, or enforce `disabled_algorithms={"keys": ["rsa-sha1"]}` in paramiko Transport config.

### CVE-2025-69872 · diskcache 5.6.3 · HIGH
- **Fix versions:** None available
- **Description:** DiskCache (python-diskcache) through 5.6.3 uses Python `pickle` for serialization by default. An attacker with write access to the cache directory can inject arbitrary pickled objects, leading to Remote Code Execution (RCE) upon deserialization.
- **Scope:** Cache subsystem — mitigated in `_secure_cache_dir()` (documented in `requirements.txt`)
- **Status:** ✅ Partially mitigated — `_secure_cache_dir()` restricts cache directory permissions. Full remediation pending upstream fix.
- **Workaround:** Ensure cache directory is not world-writable; consider `DiskCache(serializer=json)` for non-binary data.

## Recommendations (優先度順)

1. **[HIGH P1] paramiko CVE-2026-44405** — `server_home.py` Transport 設定に `disabled_algorithms={"keys": ["rsa-sha2-256", "rsa-sha2-512"]}` ではなく SHA-1 を明示的に無効化。または paramiko を commit `a448945` 以降の git snapshot に差し替え。upstream リリースを監視。

2. **[HIGH P1] diskcache CVE-2025-69872** — `_secure_cache_dir()` のパーミッション制御が機能していることを CI で verify するテストを追加。非バイナリデータには `DiskCache(serializer=json)` へ切替を検討。upstream 5.6.4+ リリースを監視。

3. **[MEDIUM P2] server_home.py:265 B601** — `exec_command(cmd)` の `cmd` 引数を allowlist でサニタイズ (例: `ALLOWED_CMDS = frozenset({"systemctl status ...", "journalctl ..."})`). 現在の呼び出しパスを確認し、ユーザー入力が混入しないことを保証。

4. **[MEDIUM P3] conversation_search.py B608** — テーブル名・カラム名を identifier allowlist で検証するユーティリティ関数を導入。SQLite は `parameterized queries` でカラム名を bind できないため、明示的な allowlist が必須。

5. **[LOW P4] cryptography 環境バージョン不一致** — 実行環境の `cryptography 41.0.7` は `requirements.txt` の `>=46.0.7` を満たしていない。`pip install -r requirements.txt` を再実行し環境を同期。CI の依存キャッシュを無効化して確認。
