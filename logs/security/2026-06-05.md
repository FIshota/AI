| 項目 | 内容 |

| CVE | CVE-2026-44405 |
| GHSA | GHSA-r374-rxx8-8654 |
| パッケージ | paramiko 3.5.1 |
| 修正バージョン | なし（upstream 対応待ち） |
| 説明 | Paramiko through 4.0.0 の `rsakey.py` が SHA-1 アルゴリズムを許可している。SHA-1 は NIST 廃止済み。RSA鍵交換・署名で cryptographic downgrade attack が可能。 |

## Recommendations (優先度順)

1. **[HIGH・即対応] paramiko CVE-2026-44405**: upstream の修正コミット `a448945` をフォロー。`requirements.txt` の `paramiko<4.0` 制約を一時解除し、パッチ版が出たらアップグレード。暫定措置として SSH 接続コード全体で `disabled_algorithms={"pubkeys": ["rsa-sha2-256-cert-v01@openssh.com", "rsa-sha2-512-cert-v01@openssh.com"]}` ではなく SHA-1 RSA を無効化。

2. **[HIGH・計画的対応] diskcache CVE-2025-69872**: `_secure_cache_dir()` 緩和は有効だが、`diskcache.Cache(disk=diskcache.JSONDisk)` または `FanoutCache` の serialize オプションを pickle-free に設定し、実質的なリスクをゼロ化。

3. **[MEDIUM] server_home.py:265 — 動的 paramiko exec_command**: `cmd` パラメータが呼び出し元で適切にサニタイズされているか確認。ホワイトリスト方式 (`ALLOWED_CMDS = {...}`) を検討。

4. **[MEDIUM] conversation_search.py — SQL f-string**: テーブル/カラム名は変数から構築されているが、入力元が信頼済みであることを assert またはホワイトリストで保証。将来の攻撃面拡大を防ぐ。

5. **[LOW・定期] 依存関係メジャーアップグレードのレビュー**: `duckduckgo-search 8.x`（requirements.txt は v7 止め）・`transformers 5.x`（optional 扱い）のAPI差分検証を次スプリントでスケジュール。
