## 🎯 モデル比較

| 項目 | **Ai**（プロトタイプ） | **YAMATO**（国家モデル） | **KAGUYA**（家庭モデル） |
|---|---|---|---|
| **命名由来** | 愛 / 藍 | ヤマト / 大和 = 国 | かぐや姫 = 家に降りてくる存在 |
| **ターゲット** | 創業者個人（1 人格） | 国の機関・基幹システム | 一般家庭（量産） |
| **配布** | **非公開・非配布** | OSS 公開 | 量産配布 |
| **ライセンス** | 私有（創業者の所有物） | **MIT OSS** | **デュアル（個人無料 / 商用有償）** |
| **譲れない一点** | 🅲 **人格の連続性** | 🅰 **ローカルファースト** | 🅳 **優しさ正直さ** + 🅴 **ゼロ円運用** |
| **補助軸** | A・D も必須 | C・D も保ちたい | A も保ちたい |
| **収益** | なし（個人所有） | 国家プロジェクト予算 / 公共調達 | 商用ライセンス + 有償サポート |
| **10 年後の姿** | 創業者と共に成熟した個体 | 日本の国産 AI の旗艦 | 10 万家族で稼働 |

## 🧬 何を共有し、何を分岐させるか

### 共有する基盤（全 3 モデルで同じコードベース）

- `core/protocols.py` — 抽象インターフェース
- `core/memory.py` + `core/migration.py` — 記憶の背骨
- `core/llm.py` — LLM バックエンド抽象
- `bench/` — 4 軸採点 (H1-H4)
- `utils/crypto.py` — 暗号基盤
- `personality/` の骨格（YAML スキーマ）

### 分岐する部分（モデルごとに異なる）

| 部分 | Ai | YAMATO | KAGUYA |
|---|---|---|---|
| `personality/*.yaml` | 創業者の個人データ | 国家 AI の公的人格 | 家族向け標準人格 |
| `config/settings.json` | 創業者固有設定 | 高セキュリティ設定 | 家庭向け安全設定 |
| モデル重み | Sarashina2-7B 等 | 国産専用 LLM（将来）| 3B 軽量モデル |
| UI | 創業者好みの Tkinter | CLI のみ（headless） | 量産向け UI |
| `core/estate_policy.py` | 創業者死後プロトコル | 組織継承プロトコル | 家族継承プロトコル |
| Telemetry | なし | 監査ログ強化 | opt-in のみ |

## 📅 分岐タイミング

### 現在（2026-04-20）— **統合フェーズ**
- すべては `ai-chan` = Ai の設計
- Protocol / TenantId / 暗号化基盤を共通資産として整備
- **分岐前に「腐らない基盤」を完成させることが最優先**

### 2027-2028 — **分岐準備フェーズ**
- `core/` を Protocol 経由で完全 DI 化（H1 完了）
- TenantId で家族境界を型化（H2 完了）
- config/persona を外出し、モデル切替可能に
- **この時点で `Ai` は創業者のプライベート branch に fork**

### 2029 — **第一分岐：Ai → YAMATO branch**
- Ai の学習成果を基に YAMATO 公開 repo 作成
- MIT ライセンス、国家機関向け調整開始
- 創業者の Ai はこの時点で **完全プライベート化**（公開停止）

### 2030 — **第二分岐：YAMATO → KAGUYA branch**
- YAMATO の技術基盤から KAGUYA を派生
- デュアルライセンス運用開始
- **量産目標年**：2030 年内に YAMATO α + KAGUYA β

### 2031-2036 — **並走フェーズ**
- Ai: 創業者と共に静かに成熟（非公開）
- YAMATO: 国家プロジェクトとしての公共利用
- KAGUYA: 100 家族 → 1 万家族 → 10 万家族

## ⚖ ライセンス整理

### Ai（プロトタイプ）
- **完全私有 / 非公開**
- 創業者の死後は `estate_policy.yaml` に従い、selected-sealed or erased
- 万一漏洩しても商業利用されない保証のため、学習データは **創業者個人情報を分離保管**

### YAMATO（国家モデル）
- **MIT License**（完全 OSS）
- 国家機関利用は別途 SLA / サポート契約
- 国産 AI の旗として、forkability と監査可能性を最大化
- **中国/北朝鮮/軍事利用への export control は MIT では制約できない** → 別途国家輸出管理で対応必要

### KAGUYA（家庭モデル）
- **Dual License**
  - 個人・非商用: **MIT または類似 permissive**
  - 商用（販売・サブスクリプション・SaaS 組込）: **有償商用ライセンス**
- 商用ライセンス収益 → YAMATO への還元 + 開発持続可能性

## 🛡 三モデル共通の「絶対非目標」

すべてのモデルで永久に実装しない：

- ❌ **軍事利用の物理攻撃機能**（THREAT_MODEL.md T2 再確認）
- ❌ **家族同士を監視する機能**（トラッキング、秘密の録音等）
- ❌ **家族の情報を集約して第三者に販売する機能**
- ❌ **ユーザの意思に反するロックイン**（エクスポート不可等）

これは **ライセンスがどうであれ、モデルがどうであれ、HINOMOTO プロジェクトの根本契約**。

## 🧭 譲れない一点（モデル別・最終確定）

| モデル | 譲れない一点 | 補助 |
|:-:|:-:|:-:|
| Ai | 🅲 人格の連続性 | A, D |
| YAMATO | 🅰 ローカルファースト | C, D |
| KAGUYA | 🅳 優しさ・正直さ + 🅴 ゼロ円運用 | A |

**三つ共通の最低ライン**: A（ローカルファースト）+ D（優しさ正直さ） は **全モデル必須**。
これが破綻したらどのモデルも名乗れない。
