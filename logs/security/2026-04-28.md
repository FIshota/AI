## 🚨 Critical / High Findings

### CVE-2025-69872 — `diskcache` 5.6.3

| Field | Detail |
|---|---|
| Package | diskcache 5.6.3 |
| CVE | CVE-2025-69872 |
| GHSA | GHSA-w8v5-vhqr-4h9v |
| Fix Versions | **None available** |
| Severity | **HIGH** (Arbitrary Code Execution) |

**Description:** DiskCache uses Python `pickle` for serialization by default. An attacker with write access to the cache directory can achieve **arbitrary code execution** when a victim application reads from the cache.

**Impact:** If the cache directory (`~/.cache/diskcache` or custom path) is writable by an attacker (e.g., shared filesystem, container escape, or compromised subprocess), malicious pickle payloads can be injected and executed on cache read.

**Mitigation (no upstream fix yet):**
- Ensure the cache directory has strict permissions (`chmod 700`)
- Consider disabling pickle serialization (pass `disk=diskcache.JSONDisk` or equivalent)
- Monitor for upstream fix or switch to an alternative caching library
